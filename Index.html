Mitigation Strategy Framework
01
Cryptographic Inventory & Discovery
Enumerate all cryptographic assets: certificates, keys, algorithms in use across applications, APIs, hardware, and third-party dependencies. You cannot protect what you cannot see.
Immediate Foundation
02
Crypto-Agility Architecture
Design systems so cryptographic algorithms can be swapped without rewriting applications. Abstract crypto behind interfaces. This is your long-term insurance policy.
Architecture Design
03
Hybrid Classical + PQC Deployment
Deploy hybrid modes that combine classical algorithms (RSA/ECC) with PQC algorithms simultaneously. Security is maintained if either system remains secure. Deploy in TLS, VPN, SSH now.
Deploy Now Hybrid Mode
04
Prioritize HNDL-Sensitive Data
Immediately re-encrypt long-lived, high-sensitivity data with AES-256 + Kyber KEM. Classify data by sensitivity lifetime. Anything sensitive for 10+ years needs PQC protection today.
Critical Data Security
05
PKI Modernization
Plan and execute migration of certificate authority infrastructure to support PQC signatures (Dilithium / FALCON). Issue hybrid certificates. Update revocation and validation infrastructure.
PKI Certificates
06
Supply Chain & Third-Party Assessment
Audit vendors, cloud providers, and software dependencies for quantum readiness. Include PQC requirements in procurement. Most breaches will come through the supply chain.

Technique Selection Guide
Need key exchange / encryption?
→ Use CRYSTALS-Kyber (fast, compact, NIST standard)
Need digital signatures (general use)?
→ Use CRYSTALS-Dilithium (balanced, widely supported)
Need compact signatures for IoT / bandwidth?
→ Use FALCON (smallest PQC signatures)
Need long-term root CA / ultra-conservative?
→ Use SPHINCS+ (hash-based, stateless, most conservative)
Need symmetric encryption (keep using)?
→ Use AES-256 — already quantum-resistant at this key size
Transitioning now — not ready to drop classical?
→ Use Hybrid mode: Kyber + ECDH simultaneously in TLS
